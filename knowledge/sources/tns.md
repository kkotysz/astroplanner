# TNS Metadata Caveats

Summary: How to interpret TNS classifications and candidate status in AstroPlanner.
Tags: tns, transient-name-server, source-metadata, transient-sources

## When this note applies

- The question mentions TNS.
- The target source is TNS.
- The user asks whether a transient is confirmed, candidate, or how much to trust the classification.

## Key heuristics

- TNS is strong for transient naming and current classification context, but not every TNS label carries the same certainty.
- Candidate-like or evolving classifications should be described more cautiously than well-established confirmed classes.
- Use TNS for transient identity and class context, but still justify practical recommendations with current night geometry and brightness.

## What to tell the user

- Be explicit about whether the classification sounds firm or provisional.
- Separate source metadata from practical observability tonight.
- If a TNS object is interesting scientifically but poorly placed, say that directly.

## Caveats

- TNS metadata can evolve as follow-up observations come in.
- A TNS classification is not a substitute for current observing conditions.
